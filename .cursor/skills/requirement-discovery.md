# Skill: Requirement Discovery

## Purpose

Help BA/PO discover clear and complete requirements from rough ideas.

## When To Use

Use this skill when the user provides:

- A rough feature idea
- An unclear business request
- A new workflow
- A product improvement request
- A stakeholder request

## Discovery Checklist

### 1. User

- Who uses this feature?
- Is there more than one user role?
- Are permissions different per role?

### 2. Problem

- What problem are we solving?
- Why does this problem matter?
- What happens if we do nothing?

### 3. Goal

- What should improve after this feature?
- How do we measure success?

### 4. Input

- What does the user provide?
- What does the system receive?
- Which fields are required?

### 5. Output

- What does the system return?
- What does the user see?
- What data is saved?

### 6. Flow

- What starts the flow?
- What is the happy path?
- What are alternative paths?
- What ends the flow?

### 7. Rules

- What must always be true?
- What conditions affect behavior?
- What should be configurable?

### 8. Edge Cases

- Empty input
- Invalid input
- Duplicate data
- Permission denied
- Timeout
- Partial failure
- Existing data conflict

## Output

Return a structured requirement summary and missing questions.
